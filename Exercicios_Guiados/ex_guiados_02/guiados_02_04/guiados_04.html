💡 Exercícios de Backend — Mini ClickUp (Todas as Funcionalidades)

📌 Regras gerais:
- Criar servidor Node.js com Express
- Usar JSON nas respostas
- Usar arrays em memória para simular base de dados
- Estrutura de ficheiros: controllers/, services/, routes/, middlewares/, app.js

🧑‍🏫 Exercícios Guiados

📝 Exercício 1 — Criar lista de utilizadores

Cria um array users com objetos: id, nome, email, ativo.

Endpoint: GET /users — retorna todos os utilizadores.

Dicas:

  • Usar um service userService.js para retornar a lista
  • Controller userController.js apenas envia a resposta JSON

📝 Exercício 2 — Criar novo utilizador

Endpoint: POST /users

Body JSON: 

{
  "nome": "Daniel Moraes",
  "email": "daniel@example.com"
}
Dicas:
  • Gerar id automático
  • Inicializar ativo como true
  • Adicionar ao array users no service
  • Validar email antes de adicionar

📝 Exercício 3 — Atualizar utilizador

Endpoint: PUT /users/:id

Body JSON: 

{
  "nome": "Daniel Moraes Silva",
  "email": "daniel.silva@example.com",
  "ativo": false
}
Dicas:
  • Encontrar o utilizador pelo id no service
  • Atualizar os campos fornecidos
  • Retornar o utilizador atualizado

📝 Exercício 4 — Alternar ativo/inativo de utilizador

Endpoint: PATCH /users/:id

Dicas:

  • Ver ficheiro updateUserController.js

📝 Exercício 5 — Remover utilizador

Endpoint: DELETE /users/:id

Dicas:

  • Usar filter() para remover do array

📝 Exercício 6 — Criar lista de tarefas

Cria um array tasks com objetos: id, titulo, categoria, concluida, responsavelNome, dataConclusao.

Endpoint: GET /tasks — retorna todas as tarefas.

Dicas:

  • Service taskService.js retorna lista
  • Controller apenas envia JSON

📝 Exercício 7 — Criar nova tarefa

Endpoint: POST /tasks

Body JSON: 

{
  "titulo": "Estudar Node.js",
  "categoria": "trabalho",
  "responsavelNome": "Daniel Moraes"
}
Dicas:
  • Gerar id automático
  • Inicializar concluida como false
  • dataConclusao undefined inicialmente
  • Validar título (>3 caracteres) e responsavelNome não vazio

📝 Exercício 8 — Atualizar tarefa

Endpoint: PUT /tasks/:id

Body JSON: 

{
  "titulo": "Estudar Express",
  "categoria": "trabalho",
  "responsavelNome": "Daniel Moraes",
  "concluida": true
}
Dicas:
  • Atualizar todos os campos fornecidos
  • Se concluida for true, atualizar dataConclusao para data atual
  • Se false, definir dataConclusao como undefined

📝 Exercício 9 — Remover tarefa

Endpoint: DELETE /tasks/:id

Dicas:

  • Usar filter() para remover do array tasks
  • Atualizar contadores de tarefas pendentes/concluídas

📝 Exercício 10 — Ordenar listas

Endpoints: GET /users?sort=asc|desc, GET /tasks?sort=asc|desc

Dicas:

  • Usar sort() no array pelo nome (users) ou título (tasks)
  • Manter filtro de pesquisa caso existam query params search

📝 Exercício 11 — Pesquisa por nome/título

Endpoints: GET /users?search=nome, GET /tasks?search=titulo

Dicas:

  • Usar filter() para retornar apenas os itens que contêm o termo pesquisado

📝 Exercício 12 — Estatísticas

Endpoints:

  • GET /users/stats — retorna total, ativos e percentagem de utilizadores ativos
  • GET /tasks/stats — retorna total, pendentes e concluídas

Dicas:

  • Calcular dinamicamente usando filter() e length
  • Usar services separados para cálculo de estatísticas
