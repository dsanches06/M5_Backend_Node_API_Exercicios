💡 Projeto 1 — Primeira API Completa

  • O objetivo deste projeto é criar uma API completa para gerenciamento de tarefas, incluindo utilizadores, tarefas, tags e comentários.
  • Deve ser entregue todos os endpoints mencionados em aula e nos exercícios de entrega.
  • Usar Node.js com Express e JSON nas respostas.
  • Todos os dados devem ser mantidos em memória usando arrays (simulando base de dados).
  • Validações e lógica de negócio devem ser feitas nos services, controllers apenas retornam JSON.
  • IDs devem ser gerados automaticamente para todas as entidades (users, tasks, tags, comments).

📁 Estrutura final do projeto

src/
  controllers/
    userController.js
    taskController.js        <-- também trata /tasks/:id/comments e /tasks/:id/tags
    tagController.js
  services/
    userService.js
    taskService.js           <-- array tasks e funções auxiliares
    tagService.js            <-- array tags, importa taskService para validar tarefas
    commentService.js        <-- array comments, importa taskService e userService para validações
  routes/
    userRoutes.js
    taskRoutes.js            <-- endpoints de tarefas, comentários e tags por tarefa
    tagRoutes.js             <-- endpoints globais de tags
  middlewares/
    checkUserExists.js
  app.js

Observação: Não é necessário criar arrays de comentários ou tags dentro dos controllers. Controllers chamam os services e retornam JSON.

📝 Exercícios de entrega (Novas funcionalidades)

Exercício 1 — Criar sistema de tags (atualizado)

Criar sistema de tags para classificar tarefas.

Objeto tag: 
id
nome
Endpoints: 
GET /tags
POST /tags
DELETE /tags/:id
Body JSON POST: 
{
  "nome": "backend"
}
Dicas:
  • Criar array tags dentro de tagService.js
  • Gerar id automático
  • Validar nome da tag
  • Ao deletar uma tag, remover também todas as associações dessa tag no array taskTags do taskService
  • Service retorna a tag deletada ou uma mensagem de erro, controller apenas retorna JSON

Exercício 2 — Associar tag a uma tarefa

Uma tarefa pode ter várias tags e uma tag pode pertencer a várias tarefas (N:N).

Array de relação dentro do taskService.js: 
const taskTags = [];
Objeto relação: 
taskId
tagId
Endpoint: 
POST /tasks/:id/tags
Body JSON: 
{
  "tagId": 1
}

Nota: Existe outra forma válida de implementar a associação entre tarefas e tags. Em vez de usar um array separado taskTags, poder-se-ia armazenar diretamente um array de objetos tags dentro de cada tarefa. Essa abordagem simplifica o retorno das tarefas com suas tags, mas torna a manutenção de tags globais compartilhadas entre várias tarefas um pouco mais complexa. Ambas as implementações são aceitáveis para este projeto.

Dicas:
  • Verificar se tarefa existe (importando taskService)
  • Verificar se tag existe (importando tagService)
  • Evitar duplicação da mesma associação
  • Service deve retornar a associação criada, controller apenas retorna JSON

Exercício 3 — Listar tarefas de uma tag

Endpoint: 
GET /tags/:id/tasks
Retorna todas as tarefas associadas à tag com id fornecido. Dicas:
  • Usar array taskTags do taskService para buscar associações
  • Buscar as tarefas correspondentes no array tasks do taskService
  • Retornar JSON com lista de tarefas

Exercício 4 — Criar comentários em tarefas

Comentários são 1:N com tarefas (cada comentário pertence a uma única tarefa).

Array dentro de commentService.js: 
const comments = [];
Objeto comentário:

dataCriacao

Exercício 5 — Listar comentários de uma tarefa

Endpoint: 
GET /tasks/:id/comments
Retorna todos os comentários associados à tarefa. Dicas:
  • Filtrar array comments pelo taskId
  • Ordenar por data se desejado
  • Retornar JSON
